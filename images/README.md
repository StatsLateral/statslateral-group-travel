# Image Placeholders

This directory contains placeholder references for all images needed for the app landing page. When implementing the actual website, replace these placeholder descriptions with real image files.

## Required Images

1. **logo.png**
   - Description: Your app logo
   - Recommended size: 160px × 40px
   - Used in: Header and footer

2. **app-screenshot.png**
   - Description: Main app screenshot for the hero section
   - Recommended size: 600px × 1200px
   - Used in: Hero section

3. **step-1.png**, **step-2.png**, **step-3.png**
   - Description: Icons for the "How It Works" section
   - Recommended size: 80px × 80px
   - Used in: How It Works section

4. **user-1.png**, **user-2.png**, **user-3.png**
   - Description: User avatars for testimonials
   - Recommended size: 50px × 50px (square)
   - Used in: Testimonials section

## Image Guidelines

- Use PNG format for logos and icons to maintain transparency
- Optimize all images for web to ensure fast loading times
- Maintain consistent style across all icons and graphics
- Consider using SVG format for icons when possible for better scaling

## GitHub Storage

When storing these images in GitHub:
1. Commit all images to this `/images` directory
2. Use relative paths in HTML/CSS (e.g., `src="images/logo.png"`)
3. Consider using GitHub LFS (Large File Storage) if your images exceed 5MB

## Placeholder Strategy

Until you have final images, you can:
1. Use generic placeholder services like `https://placehold.co/600x400?text=App+Screenshot`
2. Create simple colored rectangles with text labels
3. Use free stock photos with proper attribution
